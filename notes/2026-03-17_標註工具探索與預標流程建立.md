## 二、預標流程（prelabel.py）

用訓練好的 100ep `best.pt` 對新圖片先做推論，輸出 YOLO 格式的 `.txt` 標註檔，減少從零手動標記的工作量。

```
python prelabel.py
```

**結果（go-board valid 46 張）：**
- 成功預標：34 張
- 未偵測到（空標註）：12 張（色調特殊、光線差）

**輸出位置：**
```
prelabel_output/
├── images/      ← 原始圖片
├── labels/      ← labelme .json（手動標後放這）
└── yolo_labels/ ← 轉換後的 YOLO .txt
```

## 三、標註工具評估

### labelImg（不穩定）
- 安裝：`pip install pyqt5 lxml labelImg`
- 問題：按 `D` 切換圖片時視窗消失（Python 3.9 + PyQt5 相容性 bug）
- **結論：不建議使用**

### labelme（推薦）
- 安裝：`pip install labelme`
- 版本：5.10.1
- 優點：穩定、支援多種標註類型、存 JSON 格式
- 開啟方式：
```powershell
labelme "datasets\go-board-1\valid\images"
```
- 快捷鍵：`W` 畫框 / `D` 下一張 / `A` 上一張 / `Ctrl+S` 儲存

### Roboflow 線上標註（SAM3 自動偵測）
- 上傳圖片後可用 SAM3 模型自動偵測物件
- 適合快速批次預標，再人工修正
- 優點：不需要本地環境、AI 輔助標記

---

## 四、labelme JSON → YOLO txt 轉換

labelme 存的是 `.json` 格式，需轉成 YOLO 的 `.txt` 才能用於訓練。

```
python convert_labelme_to_yolo.py
```

**轉換邏輯：**
- `rectangle` 的兩個頂點座標 → 正規化的 `cx cy w h`
- 類別對應：`black=0`、`white=1`
- 輸出到 `prelabel_output/yolo_labels/`

**測試結果：**
- 1 張圖（試標）→ 14 個框框，格式正確

## 五、今日問答摘要

| 問題 | 結論 |
|------|------|
| 漏框的原因 | 信心分數低於門檻、棋子重疊、訓練資料未涵蓋該光線/角度 |
| 整張圖都沒框的原因 | Domain shift：色調/光線與訓練資料差異過大 |
| 不同大小圖片 YOLO 如何處理 | Letterbox 等比例縮放＋補灰邊，座標自動反推回原圖 |
