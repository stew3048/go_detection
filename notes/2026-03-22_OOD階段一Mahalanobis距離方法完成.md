## 二、OOD 是什麼？

**OOD = Out-of-Distribution Detection（超出已知分佈的偵測）**

OOD 不是一種特定模型，而是一個**任務/概念**，目標是：

> 「模型在不知道外星人存在的情況下，只靠看過大量正常棋子，識別出從未見過的異常物件。」

### OOD vs 一般分類的差異

| | 一般分類（YOLO）| OOD |
|--|----------------|-----|
| 問的問題 | 這是 black 還是 white？ | 這在已知範圍內嗎？ |
| 世界觀 | 閉世界：只有已知類別 | 開世界：可能有未知物件 |
| 遇到黃棋 | 硬選 black 或 white | 說「我不認識這個」|
| 需要黃棋資料 | 需要 | **不需要** |

## 三、我們的架構（兩道關卡）

```
第一關（YOLO best.pt）：有沒有東西？在哪裡？
        ↓ 找出框框，截圖
第二關（OOD 分析）：這個截圖是已知的還是未知的？
        ↓ 正常 → black / white
        ↓ 異常 → ALIEN ⚠️（紅框標記）
```

**YOLO 用的就是之前訓練好的 `best.pt`（100ep）**，負責找出棋子位置，OOD 負責判斷每顆棋子是否異常。

## 四、Mahalanobis Distance 方法流程

### Step 1：合成外星棋子（synthesize_alien.py）
- 取測試集 10 張圖片，每張合成 2 顆黃色圓形棋子
- 黃棋有高光效果，視覺上接近真實棋子
- 放置時自動避開已有棋子的位置
- 輸出：`runs/ood_test/alien_*.jpg`（有黃棋）+ `normal_*.jpg`（對照組）

### Step 2：建立正常棋子的特徵分佈（ood_mahalanobis.py）
- 用 `best.pt` 對訓練集 100 張圖推論
- 每個偵測到的框框截圖 → 提取 **HSV 色彩直方圖特徵向量（32 維）**
- 分別計算 black（976 顆）和 white（886 顆）的：
  - mean（平均特徵向量）
  - covariance matrix（共變異數矩陣）
  - inverse covariance（計算距離用）

### Step 3：推論時做 OOD 判斷
- 新框框截圖 → 提取 HSV 特徵（32 維）
- 計算與 black 分佈的 Mahalanobis Distance
- 計算與 white 分佈的 Mahalanobis Distance
- 取最小值（代表「最像哪個已知類別」的距離）
- 距離 > 7.5 → **ALIEN**（紅框 + 顯示距離值）

## 五、HSV 特徵向量是什麼？

HSV 是一種色彩空間：
- **H（Hue）色相**：顏色是什麼（0~180）
- **S（Saturation）飽和度**：顏色純不純（0=灰, 255=鮮豔）
- **V（Value）亮度**：明不明亮（0=黑, 255=白）

各類棋子的 HSV 特徵：
```
黑棋：S 低（接近灰），V 低（很暗）
白棋：S 低（接近灰），V 高（很亮）
黃棋：H ≈ 30（黃色色相），S 高（鮮豔）← 與黑白棋完全不同
```

**直方圖**：統計截圖裡每個 HSV 值出現幾次，分成 H=16格、S=8格、V=8格，共 32 個數字組成特徵向量。

## 六、Mahalanobis Distance 是什麼？

### 誰跟誰算距離？

```
新框框的 HSV 特徵向量（32維）
    vs
訓練集黑棋的平均特徵分佈
    vs
訓練集白棋的平均特徵分佈
→ 取兩者中較小的距離
```

### 為什麼不用歐式距離？

Mahalanobis = **把每個維度先標準化，再算距離**

```
歐式距離：只管「離得遠不遠」（所有維度同等重要）
Mahalanobis：還考慮「這個方向的資料變化大不大」
```

具體舉例（單一維度 V 亮度）：
```
黑棋訓練資料 V：mean=20, std=4（很集中）
新棋子 V=128

Mahalanobis 距離 = (128 - 20) / 4 = 27 個標準差 → 極度異常

若用歐式距離：128 - 20 = 108（無法判斷這大不大）
```

**std 小（資料集中）的維度，距離被放大** → 黃棋高飽和度在黑白棋分佈裡幾乎不存在，標準差極小，所以距離被大幅放大。

公式：
```
d = √[ (x - mean)ᵀ × Σ⁻¹ × (x - mean) ]

x     = 新框框特徵向量
mean  = 訓練集平均特徵向量
Σ⁻¹   = 共變異數矩陣的逆（讓每個維度除以自己的變異數）
```

## 八、發現的限制

**YOLO 沒框到的黃棋，OOD 也偵測不到。**

- `alien_frame_00530` 和 `alien_frame_00545` 各只偵測到 1 顆黃棋（應有 2 顆）
- 第二顆黃棋 YOLO 信心分數太低（< 0.20），被過濾，OOD 連截圖都拿不到
- 原因：best.pt 從未見過黃棋，對黃棋信心本來就低，位置稍微不利就被丟棄

**這正是下一步做 AutoEncoder 的動機：**
AutoEncoder 分析整張圖片，不依賴 YOLO 先找位置，黃棋不管信心多低都能被偵測到。
