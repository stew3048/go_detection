# 2026-03-19｜OOD 外星棋子偵測概念討論與實驗規劃

## 一、起源想法

> 「如果在黑棋白棋的棋盤中，混入一顆黃棋，YOLO 抓得到嗎？他怎麼知道那是外星人？」

這個問題引出了一個核心挑戰：**模型在不知道外星人存在的情況下，只靠看過大量正常棋子，能否識別出從未見過的異常物件？**

---

## 二、為什麼標準 YOLO 做不到

標準 YOLO 是分類模型，輸出只有已知類別的機率：
- 遇到黃棋 → 強制在 `black` / `white` 二選一
- 沒有「我不認識這個」的選項

根本原因：
| YOLO 在做的 | 我們想要的 |
|------------|-----------|
| 學 black vs white 的**邊界** | 學什麼是**正常棋子的分佈** |
| 閉世界假設（只有已知類別）| 開世界假設（可能出現未知物件）|

---

## 三、OOD 偵測概念（Out-of-Distribution Detection）

> 「看過很多非外星人，所以應該要知道沒看過的就是外星人」

架構：
```
圖片輸入
    ↓
YOLO 偵測（找出所有疑似棋子的框框）
    ↓
每個框框截圖 → 送進 OOD 分析模組
    ↓
像訓練時看過的棋子 → black 或 white ✅
不像 → 外星人 ⚠️（標紅框 + ALIEN）
```

---

## 四、預計實驗計畫（兩階段）

### 階段一：Mahalanobis Distance（先做，驗證可行性）

**原理：**
1. 把訓練集所有棋子框框截圖，送進 YOLO backbone 取得特徵向量
2. 分別計算 black、white 的特徵分佈中心（mean）與共變異數矩陣
3. 推論時，新框框的特徵向量與兩個分佈中心都很遠 → 外星人

**優點：** 實作簡單、速度快、不需額外訓練  
**缺點：** 對特徵空間的線性假設，可能漏抓複雜的異常

**預計輸出：**
- 每個框框的異常分數（距離值）
- 設定門檻，超過 → 標記 `ALIEN`（紅框）

---

### 階段二：AutoEncoder（後做，更有趣）

**原理：**
```
訓練階段：
  只餵正常棋子圖片（black + white）
  AutoEncoder 學會「壓縮 → 還原」正常棋子
  
推論階段：
  黑棋/白棋 → 還原誤差小 ✅（模型認識）
  黃棋      → 還原誤差大 ⚠️（模型沒見過，重建失敗）
```

**為什麼有趣：**
- 不依賴 YOLO 的特徵，完全自學「正常長什麼樣」
- 誤差圖可視化，能直觀看出哪裡「重建失敗」
- 概念上更接近人類直覺：「這個我看不懂，畫不出來」

**架構（預計）：**
```
輸入：截圖的棋子小圖（64×64）
Encoder：Conv → Conv → 瓶頸層（latent vector）
Decoder：ConvTranspose → ConvTranspose → 輸出重建圖
損失函數：MSE（原圖 vs 重建圖）
異常判斷：重建誤差 > 門檻 → ALIEN
```
